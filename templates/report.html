{% if p.port == 21 %}FTP (File Transfer Protocol) allows uploading and downloading files to/from the server. It was designed before internet security was a concern and has no built-in encryption — everything is transmitted in plain text. {% elif p.port == 22 %}SSH (Secure Shell) is an encrypted protocol for remotely accessing and managing a server over the internet. It is the standard, secure replacement for Telnet. {% elif p.port == 23 %}Telnet is a legacy protocol for remote server access. It was created in 1969 when security was not a concern. It transmits absolutely everything — commands, responses, and passwords — as plain readable text. {% elif p.port == 25 %}SMTP (Simple Mail Transfer Protocol) is the standard protocol for sending emails between mail servers. Port 25 handles server-to-server email delivery across the internet. {% elif p.port == 53 %}DNS (Domain Name System) is the internet's phone book — it translates domain names like fupre.edu.ng into IP addresses. It runs on port 53 and handles billions of queries daily. {% elif p.port == 80 %}HTTP (HyperText Transfer Protocol) is the foundation of web communication. Port 80 serves web pages, but with zero encryption — all data travels in plain readable text. {% elif p.port == 110 %}POP3 (Post Office Protocol) is used by email clients to download emails from the server to a local device. Without SSL, credentials and email content are transmitted unencrypted. {% elif p.port == 143 %}IMAP (Internet Message Access Protocol) allows email clients to access and manage emails stored on the mail server. Without encryption, all email access credentials and content are exposed. {% elif p.port == 443 %}HTTPS (HTTP Secure) is the encrypted version of HTTP. 💥 What could an attacker do?
{% if p.port == 21 %}An attacker uses a packet sniffer (free tool) to capture all FTP network traffic and extract your login username and password in plain text. They can then log in with your credentials and have complete control of the server. {% elif p.port == 25 %}If misconfigured as an open relay, attackers route millions of spam and phishing emails through your server without authentication. Your IP gets blacklisted, your domain reputation is destroyed, and all legitimate emails from your domain start going to spam folders. {% elif p.port == 53 %}Attackers exploit misconfigured DNS servers for DNS amplification attacks (sending massive traffic floods to victims), DNS cache poisoning (redirecting your users to fake websites controlled by the attacker), or zone transfer attacks (downloading your complete internal network map). {% elif p.port == 80 %}Using tools like Wireshark or mitmproxy, attackers intercept unencrypted HTTP traffic to: steal session cookies and take over user accounts without needing passwords, capture login form data in transit, modify web page content being delivered to users (injecting malicious scripts or fake content). {% elif p.port == 110 %}Attackers intercept POP3 traffic and extract email login credentials in plain text.
